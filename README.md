## 정책

1. 깃허브를 통해 Product의 이력을 관리하고 개발자 개별의 생산성을 측정한다.
2. 모든코드는 Head of Product 또는 동료 리뷰를 실시한다.
3. 매일 아침 업무 시작전 주간목표의 진행상항를 체크하고 20분내외로 스크럼 회의(이슈, 어제작업(회고), 오늘할일)를 실시한다.
